## 1. 房源管理现状与痛点

| 优先级 | 痛点 | 说明 |
|--------|------|------|
| **P0** | **状态不实时** | 空置/在租/整备/招租切换滞后，管理者无法回答「这套房此刻在生命周期哪一段」 |
| **P1** | **维护记录碎片化** | 巡检、维修、保洁分散在表格或多系统，缺 **单房源时间轴** |
| **P2** | **评分标准不统一** | 会议期望：改造前后 **按房间、按维度** 打分（如改造前 3.5、后 8.5），线上未统一 |

会议原话要点：每间房多维度打分、细项可展开，**「清清楚楚才叫管到一套房」** — 见 [AI应用于管理模式规划 (1).txt](meeting/AI应用于管理模式规划%20(1).txt)。

## 2. 房源全生命周期画像

```mermaid
flowchart LR
  S1[收房] --> S2[整备]
  S2 --> S3[招租]
  S3 --> S4[在租]
  S4 --> S5[续约或退租]
  S5 --> S3
```

| 阶段 | 关键指标（示例） | NFC / 打卡节点（与 PRD 对齐） |
|------|------------------|-------------------------------|
| 收房 | 交割时效、瑕疵率 | 可选现场节点记录 |
| 整备 | 工期、超期天数、验收一次通过率 | 资管 **整备监督 / 装修验收** 打卡 |
| 招租 | 挂牌天数、带看量、空看率 | 经纪人 **空看**、管家 **带看** |
| 在租 | 出租率、欠费、报修闭环 | 管家 **巡检**、维修关联房源 |
| 续约/退租 | 续约率、退租验房 | 验房打卡与照片 |

## 3. 房源数字化评分模型（建议）

参照会议「装修前后评分」：**按房间 / 公共区域** 设维度，改造前/后各打一次或多次复诊。

| 维度 | 说明 | 权重（示例，可调） |
|------|------|---------------------|
| **整洁度** | 卫生与杂物 | 15% |
| **设施完好度** | 家电家具可用 | 20% |
| **装修品质** | 材料与做工 | 25% |
| **安全性** | 电气燃气、防滑等 | 25% |
| **区位与采光** | 客观项+可楼层修正 | 15% |

**规则**：每维度分档打分 + 证据（照片、NFC 到位）；**总分 = Σ(维度分×权重)**；展示 **改造前/后分差** 与 **加分项清单**。

**AI 增强**：图像对比辅助 **裂缝/污渍/损坏** 检出（选型见 [研讨关键问题.md](研讨关键问题.md) 技术类 Q15）。

## 4. AI 赋能的智能房源画像

| 层级 | 能力 | 数据输入 |
|------|------|----------|
| **感知** | 实时状态、围栏到房 | `nfc_tags.gps` + `attendance_records.house_id` |
| **数据** | 带看热度、转化率、租金坪效 | 业务与签约系统 |
| **预测** | 空置风险、租金建议、维修风险 | 历史成交 + 巡检文本 + 外部比价（合规） |

## 5. 房源画像与人员画像的关联

| 角色 | 与房源关系 | 典型画像指标 |
|------|------------|-------------|
| **资管经理** | 收房与整备责任 | 单套整备周期、验收退回次数、收房质量分 |
| **租务管家** | 在租运营与招租 | 负责房源 **巡检覆盖率**、**维修响应**、带看转化 |
| **经纪人** | 渠道带看 | 空看次数、带看转定 |

人员 **画像下钻** 应能链到 **房源列表**；房源 **画像下钻** 应能链到 **时间轴上各角色动作**。

## 6. 房源看板指标设计

### 6.1 单房源看板

- 状态机当前态、评分与趋势、NFC 标签 ID  
- 打卡与巡检记录列表、维修/保洁摘要  
- 关联资管/管家/经纪人（来自 `house_assignments` 类逻辑）

### 6.2 区域看板

- 空置率、均价带、整备超期套数、带看热力（脱敏聚合）

### 6.3 管理者看板

- **问题房源预警**（长期空置、高退租、低分反复）  
- **资源配置建议**（试点阶段可规则化，后续模型化）
